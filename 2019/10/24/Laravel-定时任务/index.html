Laravel-定时任务

Laravel

摘要

在我们项目当中，有时候需要定时去处理一些特定的业务。过去，你可能需要在服务器上为每一个调度任务去创建 Cron 入口。但是这种方式很快会变得不友好，因为这些任务调度不在源代码中，并且你每次都需要通过 SSH 链接登录到服务器中才能增加 Cron 入口。

Laravel 命令行调度器允许你在 Laravel 中对命令调度进行清晰流畅的定义。且使用这个任务调度器时，你只需要在你的服务器上创建单个 Cron 入口。你的任务调度在 app/Console/Kernel.php 的 schedule 方法中进行定义。

Kernel.php

不管我们是使用定时任务或者其他需要终端执行的程序，在 Laravel 当中都要在 app\console\kernel.php当中的 commands 引入我们自定义的 artisan 命令类，schedule 执行我们的定时任务。

# 加载 artisan 自定义命令类
protected $commands = [
        // 引入自定义的终端命令类
    ];    
# 执行 定时任务    
protected function schedule(Schedule $schedule)
{
    # 简单使用
    $schedule->call(function(){
        Log::info('记录的日志',['ip' => request()->path()]);
    })->daily();
    $schedule->call('App\Services\xxxService@index')->everyMinute();
    # 执行 command
    $schedule->command('order:start')
              ->hourly();
}
# 注册 artisan 执行的命令，对应的也就是我们 routes 下面的 console.php
protected function commands()
{
    $this->load(__DIR__.'/Commands');

    require base_path('routes/console.php');
}

在 Laravel 5.5 版本以后，无需在 commands 引入。

创建 Command

php artisan make:command AddOrderCount

当创建完成 command 之后，按自己需求修改 command

class OrderCount extends Command
{
    // 执行的 artisan 命令
    protected $signature = 'order:start';
    // artisan 命令的描述
    protected $description = 'day order count data';

    public function __construct()
    {
        parent::__construct();
    }
    // 这里即可执行 业务逻辑的处理
    public function handle()
    {
        // 要执行的 处理业务逻辑
    }
}

执行 Crontab

1、创建 crontab 执行的文件
echo "* * * * * /usr/local/bin/php /var/www/html/vpay/artisan schedule:run >> /dev/null 2>&1" > order.txt
2、添加到 crontab 当中
crontab order.txt
3、开启 crontab
crontabl -l 
4、关闭 crontab 全部清除
crontabl -r

在 Linux 创建一个 Crontab 执行文件时，所有的执行命令加上绝对路径！
