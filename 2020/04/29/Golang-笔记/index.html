摘要

Go 是一门全新的语言。尽管它从现有的语言中借鉴了许多理念，但其与众不同的特性， 使得使用 Go 编程在本质上就不同于其它语言。做为 PHP 开发人员，第二门语言选 Go 的同撩还是比较多，从此可以论证 Go 还是非常值得学习！

安装

1、下载
# 从 Go 语言中文网：https://studygolang.com/dl 根据自己系统下载 Go 软件包
sudo tar -xzf go1.13.6.linux-amd64.tar.gz -C /usr/local
2、建立软链接
sudo ln -s /usr/local/go/bin/* /usr/bin/
3、测试
go version
4、配置环境变量
# Linux 如果是zsh,就把/.bashrc 改成 /.zshr
sudo vim ~/.bashrc

export GOROOT=/usr/local/go   # 表示 Go 在你的电脑上的安装位置，有些安装包会自动设置默认的 goroot
export GOPATH=$HOME/go-work   # 默认安装包的路径,把默认路径换成自己的路径
export PATH=$PATH:$GOROOT/bin:$GOPATH/bin

# 如果没有出现问题，那么就算装好了。接下来就是设置 Go 的开发环境了

# 使配置文件生效，这里有个大坑，千万别用 fish 里面执行它一直报错，人都差点搞死了
source ~/.bashrc　

# mac os

GOPATH 现在的 Go 语言来说不晓得也没关系，现在官方推荐 Go module 跟 Composer 有点相像！

Go Module

Go module 是 Go 官方自带的 Go 依赖管理库（感觉跟 Composer 很像），在 1.13 版本正式推荐使用。Go module 可以将某个项目(文件夹)下的所有依赖整理成一个 go.mod 文件，里面写入了依赖的版本等。使用 Go module 之后我们可不用将代码放置在 src下了。

1、开启 Go module，开启代理
# 设置环境变量，开启 Go module
export GO111MODULE=on

# 换成国内代理，如果不换，你会体验到什么是超级漫长的等待...
export GOPROXY=https://goproxy.cn
2、初始化项目
# 随便在任何一个位置创建一个新的项目
mkdir learn

# 初始化,会在当前目录下生成一个 go.mod 文件并且为空
go mod init learn

# 检查依赖，tidy 会检测该文件夹目录下所有引入的依赖,拉取缺少的模块，移除不用的模块
go mod tidy

当然，可以不用 tidy 检查依赖，直接 go build main.go 或者 go run main.go 也会自动把依赖导入好，所以 go module 你值得拥有官方都在推了，那还等什么用起来。

3、导入依赖
# 导入依赖
go mod vendor

# 执行此命令，会将刚才下载至 GOPATH 下的依赖转移至该项目根目录下的 vendor(自动新建) 文件夹下

4、升级
# 将会升级到最新的次要版本或者修订版本(x.y.z, z是修订版本号， y是次要版本号)
go get -u 

# 将会升级到最新的修订版本
go get -u=patch 

# 将会升级到指定的版本号 version
go get package@version
5、常用命令
go mod init  # 初始化 go.mod
go mod tidy  # 更新依赖文件
go mod download  # 下载依赖的 module 到本地 cache
go mod vendor  # 将依赖转移至本地的 vendor 文件
go mod edit  # 手动修改依赖文件
go mod graph  # 打印依赖图
go mod verify  # 校验依赖

国际惯例

1、你好世界
package main

import "fmt"

func main (){
    fmt.Println("你好世界")
}

Go 语言的代码通过包（package）组织，包类似于其它语言里的库（libraries）或者模块（modules）。也就是说程序一开始，就要声明这个程序属于哪个包。一个包由位于单个目录下的一个或多个.go源代码文件组成, 目录定义包的作用。每个源文件都以一条package声明语句开始，紧跟着一系列导入（import）的包，之后是存储在这个文件里的程序语句。

2、特殊的 main 包

main 包比较特殊，他定义了一个独立可执行的程序，他不是一个库！他是整个程序执行的入口。main 包当中的 main 函数就是整个程序执行的入口了，也就是 main 主函数，一般 main 主函数当中会调用其它很多函数完成工作。

3、需要遵循的规定

首先要注意一点，package 先 ，import 后，他们不能颠倒位置，package 只能有一个而不能多个，import 可以有多个。其次如果说 import 导入多个包，那么你都必须去使用导入的包，不然编译无法通过，这正是遵循了 Go 的格言：“没有不必要的代码！“。Go 的语法要求特别的严格，在 package 和 import 之后，才可以定义 函数、常亮、变量、类型的声明语句。程序语句他们没有顺序要求，可以随心所欲来！Go 语言不需要在语句或者声明的末尾添加分号，除非一行上有多条语句。如：for，if 等。实际上，编译器会主动把特定符号后的换行符转换为分号, 因此换行符添加的位置会影响 Go 代码的正确解析。

4、编译程序

Go 是一门编译型语言，Go 语言的工具链将源代码及其依赖转换成计算机的机器指令（译注：静态编译）。 Go 语言提供的工具都通过一个单独的命令 Go 调用，Go 命令有一系列子命令。最简单的一个子命令就是 run。这个命令编译一个或多个以 .go 结尾的源文件，链接库文件，并运行最终生成的可执行文件。注意：Go 语言编译代码是一行一行的编译，所以一行只能写一句代码，不能一行写多句代码，否则报错！

# 编译并运行源文件
go run hello.go

编译生成一个可执行文件，在 Windows 下会生成一个 .exe文件，OS X 和 Linux 下会成才一个二进制文件，可以直接运行，Go 编译后不依附环境，但是需要相同的平台才能运行。

go build hello.go

如果使用了 Go module 那么go build main.go 或者 go run main.go 会自动把依赖导入好。

Go 转义符

常用的转义字符有以下几个:

  1. \t: 表示一个制表符(tab), 通常可以使用它进行排版;
  2. \n: 表示一个换行符;
  3. \: 表示一个反斜线;
  4. ": 表示一个”;
  5. \r: 表示一个回车(从当前行的最前面开始输出,覆盖以前的内容);

程序结构

命名

Go 语言中的函数名、变量名、常量名、类型名、语句标号和包名等所有的命名，都遵循一个简单的命名规则：一个名字必须以一个字母（Unicode字母）或下划线开头，后面可以跟任意数量的字母、数字或下划线。

Go 语言中类似 if 和 switch 的关键字有25个；关键字不能用于自定义名字，只能在特定语法结构中使用。

break      default       func     interface   select
case       defer         go       map         struct
chan       else          goto     package     switch
const      fallthrough   if       range       type
continue   for           import   return      var

此外，还有大约 30 多个预定义的名字，比如 int 和 true 等，主要对应内建的常量、类型和函数。

内建常量:  true false iota nil

内建类型:  int int8 int16 int32 int64
          uint uint8 uint16 uint32 uint64 uintptr
          float32 float64 complex128 complex64
          bool byte rune string error

内建函数:  make len cap new append copy close delete
          complex real imag
          panic recover

这些内部预先定义的名字并不是关键字，你可以在定义中重新使用它们。在一些特殊的场景中重新定义它们也是有意义的，但是也要注意避免过度而引起语义混乱。

当标识符（包括常量、变量、类型、函数名、结构字段等等）以一个大写字母开头，如：Echo，那么使用这种形式的标识符的对象就可以被外部包的代码所使用（客户端程序需要先 import 导入这个包），这被称为导出（像面向对象语言中的 public ）；标识符如果以小写字母开头，则对包外是不可见的，但是他们在整个包的内部是可见并且可用的（像面向对象语言中的 private ）。

名字的长度没有逻辑限制，但是 Go 语言的风格是尽量使用短小的名字。如果一个名字的作用域比较大，生命周期也比较长，那么用长的名字将会更有意义。Go 在习惯上推荐使用驼峰命令法。

变量

Go 是一门强类型语言，但是 Go 也提供了隐式声明，在我的认知当中，只要声明时没有类型只有表达式，那么就属于隐式声明。

var 声明变量

语法：类型和表达式可以省略其中一个，但不能二个都省略。

var 变量名字 类型 = 表达式

如果省略的是类型信息，那么将根据初始化表达式来推导变量的类型信息。

// 隐式声明
var a = 1

fmt.Print(a,reflect.TypeOf(a)) // 1 int

如果初始化表达式被省略，那么将用零值初始化该变量。Go 在声明变量时，内存当中就已经对这个变量进行了初始化操作。数值类型变量对应的零值是 0，布尔类型变量对应的零值是 false，字符串类型对应的零值是空字符串，接口或引用类型（包括 slice、指针、map、chan 和函数）变量对应的零值是 nil。数组或结构体等聚合类型对应的零值是每个元素或字段都是对应该类型的零值。

var a string
var b int
var c float32

fmt.Print(a,b,c) // ""、0、0

零值初始化机制可以确保每个声明的变量总是有一个良好定义的值，因此在 Go 语言中不存在未初始化的变量。这个特性可以简化很多代码，而且可以在没有增加额外工作的前提下确保边界条件下的合理行为。另外需要注意一点，当声明一个变量，那么当前程序中就必须去使用这个变量，否则报错。还是 Go 的格言：没有不必要的代码。

Go 声明变量非常多元化，除了以上声明变量的方法，还有以下声明变量的方法：

// 一组声明多个，省略表达式
var a,b,c int // 0、0、0 int int int

// 一组声明多个，省略类型
var a,b,c = 1,"你好世界",3.1415 // int string float64

// 分组声明
var (
    a int
    b = "你好世界"
    c = 3.1415
    d ,e bool = true,false
)  
// int string float64 bool bool

函数返回值初始化变量，注意：在包级别声明的变量会在 main 入口函数执行前完成初始化，局部变量将在声明语句被执行到的时候完成初始化。

var f, err = os.Open(name) // 返回一个文件和一个错误
简洁声明变量

Go 语言当中除了 var 来声明变量，还可以使用 :=来声明变量，更加的简洁。我也喜欢把 :=叫做隐式声明变量。

语法：不能省略任何的东西，因为它已经是最简洁的声明方式了。请记住:=是一个变量声明语句，而=是一个变量赋值操作。

变量名字 := 表达式

它也支持一组多个声明，但是没有分组声明。

a,b,c := 1,"你好世界",false

它也可以使用函数返回值初始化变量。

f, err := os.Open(name) // 返回一个文件和一个错误

简洁声明还有个奇妙的地方，简短变量声明左边的变量可能并不是全部都是刚刚声明的。如果有一些已经在相同的词法域声明过了，那么简短变量声明语句对这些已经声明过的变量就只有赋值行为了。在下面的代码中，第一个语句声明了 in 和 err 两个变量。在第二个语句只声明了out 一个变量，然后对已经声明的 err 进行了赋值操作，var 并不可以有类似的操作。但是简短变量声明语句中必须至少要声明一个新的变量，下面的代码将不能编译通过。

in, err := os.Open(infile)
// ...
out, err := os.Create(outfile)
它们的作用域

var 声明可以在包内部声明（ import 下），简洁声明不可以在包内部声明（ import 下）。var 在包内部声明的变量全局可以使用，而简洁声明只有在其声明的内部使用。

package main

import "fmt"

var a int = 100 // 全局可以使用

func main() {
    b := 100 // 只能在 main 函数当中使用
}

var 声明和简洁声明，如果是在相同的作用域下他们不能声明相同变量，如在相同的函数体当中。如果 var 是在包内部声明，在函数体内部使用简洁声明一个相同的变量，那么此时就是赋值。相同作用域下不能存在二个相同的变量！

func main() {
    var a int = 100
    a := 10000 // 错误声明
}
// ...
package main

import "fmt"

var a int = 100 // 全局变量

func main() {
    a := 100000 // 局部变量，他们作用域完全不一样，此时正确声明
}

指针

首先要了解，值传递和引用传递，这二个的区别。

值传递又称拷贝传值，就是将一个变量的值复制一个给另外一个变量。此时二个变量的值相同，但是此时的二个变量又是互相独立的存在，操作其中任何一个变量，另外一个变量都不会受到其影响。

a := 100
b := a
a++
fmt.Println(a,b) // 结果：a = 101 ，b = 100
fmt.Println(&a,&b) // 结果可以发现他们的内存地址完全不一样，都是独立的存在

指针（引用传递）就是将一个数据的内存空间地址拷贝一个传递给另一个变量，此时二个变量的内存空间地址都指向一个地址，改变其中一个值另外一个值也会受到影响而改变。在变量前面加上一个&符号就是取这个变量的内存地址。如需要使用指针，Go 当非常简单执行要在变量前面加上 *即可表示指针，另外 Go 的指针不支持指针的运算！

a := 100
b := &a
fmt.Println(a,*b) // 结果 a = 100，b = 100
fmt.Println(&a,b) // 结果可以发现他们的内存地址完全一致
// ...
*b = *b + 100
fmt.Println(a,*b) // 结果 a = 200，b = 200
fmt.Println(&a,b) // 结果可以发现他们的内存地址完全一致

任何类型的指针的零值都是 nil。如果 p 指向某个有效变量，那么p != nil测试为真。指针之间也是可以进行相等测试的，只有当它们指向同一个变量或全部是 nil 时才相等。

var x, y int
fmt.Println(&x == &x, &x == &y, &x == nil) // "true false false"

Go 语言当中，返回函数中局部变量也是安全的，但每次返回的结果都不同。

var a = b()

func b () * int {
    a := 1
    return &a
}

fmt.Println(a)
