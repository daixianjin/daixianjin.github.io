Stripe 支付对接

摘要

因为公司项目不止支持咱国家，还支持国外，经过一些列的讨论调研，发现 Stripe 比较符合先目前的公司需求。Stripe 除了支持信用卡支付，也与阿里和腾讯都有合作，有这二个对于咱国人来说完全就可以了。信用卡对外，支付宝和微信对内，很舒服。

SDK

composer require stripe/stripe-php

为了能正常的使用，需要支持以下的扩展，这些都是 PHP 默认集成的扩展，但是还是自行检查下是否开启。

  • curl
  • json
  • mbstring

快速入门

如果使用的 Laravel 也可以看看 Laravel 提供的订阅管理包，Cashier 也非常好用，当然还是根据公司项目而定。

  • 官方文档：https://stripe.com/docs/api
  • 错误代码速查表：https://stripe.com/docs/error-codes#payment-intent-payment-attempt-failed

Products

产品相关信息 API 地址：https://stripe.com/docs/api/products，集成了产品的 CURD 相关操作。

创建商品
<?php

$stripe = new StripeClient(config('app.stripe_secret'));

// 最常用的三个参数，name 是必填的参数，image 为数组
$res = $stripe->products->create([
    'name' => $data['name'],
    'image' => $data['image'],
    'description' => $data['description']
]);
获取商品
<?php

$stripe = new StripeClient(config('app.stripe_secret'));

// 获取的是一条数据
$res = $stripe->products->retrieve('id');
更新商品
<?php

$stripe = new StripeClient(config('app.stripe_secret'));

// 非必须的情况不建议更新这类信息，慎重
$res = $stripe->products->update('id',array $pamran);
获取商品列表
<?php

$stripe = new StripeClient(config('app.stripe_secret'));

// 获取所有商品
$res = $stripe->products->all(['limit' => 5]);
删除商品
<?php

$stripe = new StripeClient(config('app.stripe_secret'));

// 慎重，慎重，慎重，不建议使用，如果非要使用，注意如果有在交易的数据不能删除，可以修改价格后再删除
$res = $stripe->products->delete('id');

Prices

产品价格相关信息 API 地址：https://stripe.com/docs/api/prices，集成了产品价格的 CURD 相关操作。

创建价格
<?php

$stripe = new StripeClient(config('app.stripe_secret'));

// 商品创建好，其实没有价格，价格只能通过如此来创建
$res = $stripe->prices->create([
   'unit_amount' => 1999,
   'currency' => 'usd',
   'recurring' => ['interval' => 'month'],
   'product' => 'prod_IfEBaRdbbauvgl',
]);
获取价格
<?php

$stripe = new StripeClient(config('app.stripe_secret'));

$res = $stripe->prices->retrieve('id');
更新价格
<?php

$stripe = new StripeClient(config('app.stripe_secret'));

// 慎重，慎重，慎重
$res = $stripe->prices->update('id',array $param);
获取价格列表
<?php

$stripe = new StripeClient(config('app.stripe_secret'));

// 获取所有商品
$res = $stripe->prices->all(['limit' => 5]);

以上二个类别的 API 其实就是对于商品的管理，非常重要的一个环节，如果在自身系统操作商品那么就必不可少要操作这二个列表的 API ，当然在 Stripe 官网管理也是可以。

支付宝

要支持支付宝付款，结帐会话必须满足以下所有条件：

  • 所有订单项的价格必须使用相同的货币。如果您有使用不同货币的订单项，请为每种货币创建单独的Checkout会话。
  • 您只能使用一次性订单项（不支持定期订阅计划）
