摘要

可以毫不夸张的来说，Git 是每个程序必备的一个技能，在团队协作开发当中，Git 的存在犹如黑暗当中的一道光照亮着我们。设想一下，如果你在一个团队开发当中，还是使用传统的 FTP 上传模式，你会多疯狂？其实很想写一段很长的话来表达，传统模式和 Git 的对比，不过一想意义不大！Git 作为当前时代下最强大的版本控制系统（当然，这里是个人的看法）强大肯定有他独特的地方！

简介

Git 是 Linux 创始人 Linus Torvalds 开源的一款分布式版本控制系统，它能帮助我们追踪代码，版本回溯等，是项目版本管理的不二选择工具（当然还是个人看法），

初始化

# 设置用户名和邮箱帐号
git config --global user.name  "your user name"
git config --global user.email "your email"

# 生成密钥，连续三下空格不如就需要密码
ssh-keygen -t rsa -b 4096 -C "your emai在l"

# 读取公钥配置，Gitee 或 Github 当中设置 SSH Key
cat ~/.ssh/id_rsa.pub

# 修改 Git 配置文件选用 SSH 连接
vim .git/config
## 修改前
[remote "origin"]
    url = https://gitee.com/xx/xxx.git
    fetch = +refs/heads/*:refs/remotes/origin/*
## 修改后
[remote "origin"]
    url = git@gitee.com:xx/xx.git
    fetch = +refs/heads/*:refs/remotes/origin/*

基本命令

# 查看状态,从中可以看出修改，添加
git status

# 抛弃修改，无法抛弃新增与修改
git checkout "抛弃文件"

# 拉取更新
git pull xx "可以选择分支"

# 上传代码
git push xx "可以选择分支"

# 查看分支
git branch

# 查看日志
git log
