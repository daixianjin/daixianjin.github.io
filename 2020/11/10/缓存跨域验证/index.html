缓存跨域验证

HTTP
HTTP
发布日期:   2020-11-10
更新日期:   2021-03-17
文章字数:   311
阅读时长:   1 分

摘要

我们都知道在前后分离的项目当中都存在 CORS 的问题，在我们每次非简单请求时，都能发现前台访问后台接口发起了二次请求，一次 options 预请求，一次正常的请求。

方案

其实这是一个正常的跨域操作，那么又不想每次都这样可以怎么做呢？ HTTP 已经给我们提供了一个报文参数，Access-Control-Max-Age 缓存机制， Access-Control-Max-Age 这个响应头表示 preflight request （预检请求）的返回结果（即 Access-Control-Allow-Methods 和Access-Control-Allow-Headers 提供的信息） 可以被缓存多久。

返回结果可以被缓存的最长时间（秒）。

  • 在 Firefox 中，上限是24小时 （即 86400 秒）。
  • 在 Chromium v76 之前， 上限是 10 分钟（即 600 秒)。
  • 从 Chromium v76 开始，上限是 2 小时（即 7200 秒)。
  • Chromium 同时规定了一个默认值 5 秒。
  • 如果值为 -1，表示禁用缓存，则每次请求前都需要使用 OPTIONS 预检请求。
  • 需要注意的是 Access-Control-Max-Age 的设置针对完全一样的 url，如果 url 加上路径参数，其中一个 url 的Access-Control-Max-Age 设置对另一个 url 没有效果。
